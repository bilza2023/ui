### 🧠 **Taleem.Help Engineering Principles (Advanced)**

1. **Go Declarative**
   → Systems should *describe intent*, not micromanage behavior.
   → Messages are what happened. Interpreters decide what to do.

2. **Minimize State: UI is Dumb, DB is a Snapshot**
   → UI is a rendering engine — it doesn’t know logic.
   → DB is not the source of truth — *flat files and Workshop are*.
   → You don’t store *reality*, you store *frames* of it.

3. **Separate Software from Plumbing**
   → Software: defines *what* a feature is.
   → Plumbing: moves, tags, filters, uploads — *not your job.*
   → Systems must be composable. Code must be disposable.

4. **Single Format for Transit → Typed at Edges**
   → Everything moves in the same envelope (messages).
   → Backend *interprets* based on `type`.
   → Frontend *renders* based on `type`.
   → Add new behavior by writing a new interpreter — not reworking pipes.
