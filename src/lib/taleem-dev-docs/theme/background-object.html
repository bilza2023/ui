🖼️ Background Object Model

The background object defines optional slide background layers: images and patterns. This object does not include background color, which is sourced from theme.bgColor.

1. Theme vs Background

  • theme.bgColor: solid color fill (rendered first)
  • background: visual texture (image/pattern)

2. Structure of Background Object

{
  backgroundImage: string | null,
  backgroundImageOpacity: number (0.0 - 1.0),
  pattern: Pattern | null
}

Either backgroundImage or pattern must be present.

3. Pattern Types

□ Grid

{
  type: "grid",
  cellWidth, cellHeight,
  lineColor, lineWidth
}

● Dots

{
  type: "dots",
  dotRadius,
  spacingX, spacingY,
  color
}

━ Lines

{
  type: "lines",
  direction: "horizontal" | "vertical",
  spacing,
  lineColor, lineWidth
}

4. Template Usage

Set backgrounds using methods from BaseTemplate:

this.setBackground(BaseTemplate.createBackground({
  backgroundImage: "sky.jpg",
  backgroundImageOpacity: 0.6,
  pattern: null
}));

5. Rendering Order

  1. theme.bgColor fill
  2. backgroundImage (if any)
  3. pattern (if any)

This order guarantees proper layering and visibility.

6. Developer Notes

  • Never hardcode background structures — always use factory helpers
  • Validate all input via the Zod schema
  • Use opacity and pattern layering for visual depth

The background model is designed for clarity and separation of visual layers — decoupling image-based design from semantic theme color control.
